## Installation

### Direct URL

Install components directly without any configuration:

```bash
npx shadcn@latest add https://nusa-ui.vercel.app/r/button.json
```

### Namespaced Registry

For a cleaner workflow, configure the NusaUI registry in your `components.json`:

```json title="components.json"
{
  "$schema": "https://shadcn.com/schema.json",
  "registries": {
    "nusa-ui": "https://nusa-ui.vercel.app/r/{name}.json"
  }
}
```

Then install components using short names:

```bash
npx shadcn@latest add nusa-ui/button
```

## Portals Setup

NusaUI uses portals for popup components like Dialog and Popover. To ensure these display above all page content, add an `isolation: isolate` style to your application root element:

```css title="globals.css"
.root {
  isolation: isolate;
}
```

This creates a separate stacking context, preventing `z-index` conflicts with other page styles.

## Prerequisites

Your project must be set up with:

- React 18+
- Tailwind CSS v4
- TypeScript (recommended)

Make sure you have initialized shadcn/ui in your project:

```bash
npx shadcn@latest init
```

## Working with LLMs

NusaUI is built on Radix UI. When working with AI assistants, you can reference the Radix UI documentation in markdown format for accurate component APIs and patterns.

See [Radix UI Primitives](https://www.radix-ui.com/primitives) for documentation.
