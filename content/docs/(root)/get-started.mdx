## Install a Component

### Direct URL

If you want the fastest path, install a component straight from the registry URL:

```bash
npx shadcn@latest add https://nusa-ui.vercel.app/r/button.json
```

### Registry Alias

For repeated installs, add the NusaUI registry to `components.json`:

```json title="components.json"
{
  "$schema": "https://shadcn.com/schema.json",
  "registries": {
    "nusa-ui": "https://nusa-ui.vercel.app/r/{name}.json"
  }
}
```

Then use the shorter alias when adding packages:

```bash
npx shadcn@latest add nusa-ui/button
```

## Portal Root

Popup components such as Dialog, Select, and Popover rely on portals. Add `isolation: isolate` to your app root so those layers render above the rest of the page:

```css title="globals.css"
.root {
  isolation: isolate;
}
```

That keeps overlay content from colliding with page-level stacking contexts.

## Before You Begin

You should already have:

- React 18 or newer
- Tailwind CSS v4
- TypeScript, if you want full type coverage

If your project has not been initialized with the CLI yet, start there first:

```bash
npx shadcn@latest init
```

## AI Workflows

When you use an assistant or agent, point it at the markdown docs and registry output instead of relying on memory. That makes install instructions, props, and patterns easier to verify against the actual source.

For Radix-level primitives and deeper API details, refer to the upstream documentation:

https://www.radix-ui.com/primitives
